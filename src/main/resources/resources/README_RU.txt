Примечание: Мы настоятельно рекомендуем вам поместить .jar-файл приложения в отдельную папку, поскольку
приложение может создавать различные папки и файлы во время работы.

##############################      КАК ИСПОЛЬЗОВАТЬ      ##############################

     После первого запуска у вас будут созданы все необходимые файлы и папки.

     После первого запуска у вас должны быть следующие папки:

                 Logs - Во время работы программы в этой папке будет создаваться журнал, содержащий отладочные сообщения и ошибки, если таковые имеются.
                 Include - папка, из которой будут взяты файлы изображений.
                 Output - папку, в которую будут сохраняться схематики.


     У вас также будет файл под названием 'settings.json'. В нем хранятся различные настройки, которые будут использоваться при создании схемы.

     Чтобы создать схему, поместите файл изображения в папку 'Include' и запустите приложение.
     После завершения работы в папке 'Output' появится nbt-файл с тем же
     именем, что и у изображения. Приложение также может создать несколько схем,
     если в папку 'Include' поместить несколько изображений.

########################################################################################

###############################      НАСТРОЙКИ      ###############################

     Вы можете настроить различные параметры, которые будут использоваться при запуске, с помощью файла 'settings.json'.


     Все настройки находятся в файле 'settings.json':

                 inFolder - имя папки, из которой будет взято изображение для создания схемы (по умолчанию - "Include").
                 outFolder - имя папки, в которую будет сохранен файл схемы (по умолчанию - "Output").
                 DataVersion - Версия данных о содержимом схемы (лучше не трогать) (по умолчанию 3700)
                 SizeScale - Множитель размера схемы по отношению к входному изображению. (по умолчанию 1,0)
                 DisableSaveAirData - Отключает сохранение данных о положении воздушного блока, тем самым уменьшая размер файла. (по умолчанию - true)
                 MaxThreads - Максимальное количество потоков, которые могут быть созданы.
                               Чем больше потоков, тем больше схем можно обрабатывать одновременно.
                                Стоит учесть, что это также влияет на потребление процессора программой. (default is 4)
                 palettes - список палитр, которые будут использоваться для создания схем (по умолчанию это палитра обсидиана).


                 Структура палитры:
                 {
                       "BlockName": "",
                       "Min Red": 0/255,
                       "Min Green": 0/255,
                       "Min Blue": 0/255,
                       "Max Red": 0/255,
                       "Max Green": 0/255,
                       "Max Blue": 0/255,
                       "Priority": 0-infinity
                 }

                 "BlockName" - название блока minecraft (например, minecraft:cobblestone).
                 "Priority" - Приоритет палитры. Если несколько палитр удовлетворяют условиям, будет выбрана палитра с
                               более высоким приоритетом или самая последняя (если их приоритеты одинаковы).
